# Explicit Markup & Targets

Explicit markup blocks start with `.. ` followed by whitespace. Indentation is required for subsequent lines.

## Footnotes & Citations
- **Footnotes:** `.. [1]`, `.. [#]`, `.. [#label]`, `.. [*]`. Labels can be numbers, auto-numbered, or symbols.
- **Citations:** `.. [note]`. Use non-numeric labels. Simple reference name syntax.

## Hyperlink Targets
Hyperlink targets identify locations linked to by references.

- **External:** `.. _name: https://url.com`. URI or email address.
- **Internal:** `.. _name:`. Points to the following element. Chains point to the same element.
- **Indirect:** `.. _one: two_`. Points to another reference.
- **Anonymous:** `.. __: URL` or `__ URL`. No reference name.

## Implicit Hyperlink Targets
Generated by section titles and named hyperlink references with embedded URIs.
- Explicit targets override implicit targets of the same name.
- Duplicate implicit targets are invalidated.

## Directives
Used for extensions like images, admonitions, etc.
- **Syntax:** `.. directive:: [arguments]` followed by `:options:` and `[content]`.
- **Parts:** Arguments, options (field list), and content. Blank line required before content.

```rst
.. figure:: larch.png
   :scale: 50

   The larch.
```

## Substitution Definitions
Allows inline-compatible directives (image, replace) to be defined out of line.
- **Syntax:** `.. |name| directive:: content`.
- **References:** `|name|` is replaced by the directive's result in the text.

## Comments
Explicit markup blocks not recognized as other constructs.
- **Empty Comment:** `..` on a line by itself. Useful for terminating constructs or separating indented blocks.
